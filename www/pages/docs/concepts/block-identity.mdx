## <I18nText en="The baseline limitation" zh="原生方案的限制" />

<I18nText
  en="Traditional Markdown stores structure but does not give each block a durable ID. Without IDs, you cannot safely target a paragraph across edits."
  zh="传统 Markdown 保存了结构，但不会为每个块提供稳定 ID。没有 ID，就很难在多次编辑后精确定位同一段内容。"
/>

## <I18nText en="HybricMark approach" zh="HybricMark 的做法" />

<I18nText
  en="HybricMark generates UUID-like identifiers under `attrs.id` for core nodes (paragraphs, headings, lists, list items, and more based on extension config)."
  zh="HybricMark 会在核心节点（段落、标题、列表、列表项等）上生成 `attrs.id` 形式的唯一标识（具体类型由扩展配置决定）。"
/>

```json
{
  "type": "paragraph",
  "attrs": {
    "id": "f7652fb0-0815-4f41-9efb-b67fe7b18390"
  },
  "content": [{ "type": "text", "text": "Block-level targeting" }]
}
```

## <I18nText en="What this unlocks" zh="这能带来什么" />

- <I18nText en="Block comments and annotation anchors" zh="块级评论和批注锚点" />
- <I18nText en="Bidirectional references between blocks" zh="块与块之间的双向链接" />
- <I18nText en="Partial persistence (patch only changed blocks)" zh="局部持久化（只 patch 变更块）" />
- <I18nText en="Notion-like card extraction and linking workflows" zh="Notion 风格的摘录卡片与关联工作流" />

## <I18nText en="Implementation note" zh="实现建议" />

<I18nText
  en="Persist JSON, not rendered HTML, if your product depends on stable block identity. HTML can still be derived at render/export time."
  zh="如果业务依赖稳定块标识，优先持久化 JSON 而不是渲染后的 HTML。HTML 适合在展示/导出阶段再生成。"
/>
